# LNG市场数据采集策略优化方案

## 一、问题诊断

### 1.1 当前采集失败原因分析

| 失败类型 | 占比 | 具体原因 |
|---------|------|---------|
| 网络访问受限 | 60% | EIA、Platts、Bloomberg等权威网站被阻止 |
| 采集超时 | 25% | 300秒超时限制 |
| 页面结构问题 | 10% | 需要JavaScript渲染或登录认证 |
| 数据格式问题 | 5% | 返回HTML而非结构化数据 |

### 1.2 数据质量问题分类

| 问题等级 | 定义 | 处理方式 |
|---------|------|---------|
| 🔴 严重 | 数据缺失、来源不明、时效性>7天 | 立即重新采集 |
| 🟡 中等 | 数据为推算值、单一来源、时效性3-7天 | 标记并补充采集 |
| 🟢 轻微 | 数据完整但精度不足 | 记录优化 |

# 分层采集，每层60秒
def 分层采集(数据源列表):
    for 层级 in [权威源, 专业机构, 财经媒体, 推算]:
        for 源 in 层级:

### 2.3 自动重采机制

```python
# 数据质量检查与自动重采
def 数据质量检查(采集结果):
    检查项 = {
        '时效性': 数据日期 >= 当前日期-2天,
        '完整性': 必填字段齐全,
        '来源可信度': 来源在可信列表,
        '数值合理性': 价格在合理区间,
        '多源一致性': 如有多个来源，差异<10%
    }
    
    质量评分 = sum(检查项.values()) / len(检查项) * 100
    
    if 质量评分 < 75分:
        触发重新采集(采集结果.助理, 采集结果.指标)
    
    return 质量评分, 检查项
```

## 四、自动化质量监控

### 4.1 实时质量评分

```python
class 数据质量评分器:
    def 评分(self, 数据包):
        分数 = 0
        
        # 时效性（30分）
        if 数据包.日期 == 今天: 分数 += 30
        elif 数据包.日期 == 昨天: 分数 += 25
        elif 数据包.日期 >= 今天-3天: 分数 += 15
        else: 分数 += 5
        
        # 完整性（30分）
        必填字段 = ['价格', '涨跌', '来源', '日期']
        缺失数 = sum(1 for f in 必填字段 if not 数据包.get(f))
        分数 += (len(必填字段) - 缺失数) / len(必填字段) * 30
        
        # 来源可信度（20分）
        可信源 = ['EIA', 'GIE', 'Platts', 'Bloomberg', 'Reuters', 
                 'Mysteel', '隆众资讯', '生意社', 'OilPrice.com']
        if 数据包.来源 in 可信源: 分数 += 20
        elif 数据包.来源 in 次可信源: 分数 += 10
        else: 分数 += 5
        
        # 数值合理性（20分）
        if  self._价格合理性检查(数据包): 分数 += 20
        else: 分数 += 5
        
        return 分数
```

### 6.3 多源交叉验证

```python
def 交叉验证(数据源列表, 容忍度=0.1):
    """
    多个来源数据对比，差异超过容忍度则标记
    """
    价格列表 = [d['价格'] for d in 数据源列表 if d['价格']]
    if not 价格列表:
        return None, '无有效数据'
    
    平均价格 = sum(价格列表) / len(价格列表)
    最大偏差 = max(abs(p - 平均价格) / 平均价格 for p in 价格列表)
    
    if 最大偏差 > 容忍度:
        return 平均价格, f'差异过大({最大偏差:.1%})'
    
    return 平均价格, '验证通过'
```
