# Claude Code 最佳实践 Skill

## 简介

本Skill封装了Claude Code的核心工作方法论，包括Agentic Loop、Subagent协作模式、Explore-Plan-Code工作流等最佳实践。适用于需要高质量、可验证、可维护的代码生成和数据分析任务。

---

## 🎯 核心方法论

### 1. Agentic Loop（智能体循环）

```
Gather Context（收集上下文）
    ↓
Take Action（执行行动）
    ↓
Verify Results（验证结果）
    ↓
[循环直到完成]
```

**关键原则**：
- 不假设，先探索
- 每个行动后验证
- 错误时回退并调整

### 2. Subagent 协作模式

**为什么使用Subagent**：
- 隔离长任务，避免污染主会话上下文
- 并行处理多个独立任务
- 每个Subagent有独立的token预算和思考空间

**使用场景**：
- 数据采集（多个数据源并行）
- 多文件代码生成
- 复杂分析任务分解

**调用方式**：
```bash
# 创建Subagent执行独立任务
sessions_spawn \
  --task "具体任务描述" \
  --runtime "subagent" \
  --mode "run" \
  --timeoutSeconds 300
```

### 3. Explore-Plan-Code 工作流

#### Phase 1: Explore（探索）
- 读取现有代码/数据
- 理解系统架构
- 识别依赖关系
- **禁止**：直接开始编码

#### Phase 2: Plan（规划）
- 制定执行计划
- 分解任务步骤
- 识别风险和依赖
- 输出：结构化计划文档

#### Phase 3: Code（编码）
- 按规划执行
- 小步提交，频繁验证
- 保持代码整洁

### 4. Verification-First（验证优先）

**多源交叉验证**：
- 同一指标至少2个独立来源
- 差异>5%需标注并调查
- 优先使用权威数据源

**时效性检查**：
- 标注数据时间戳
- 区分实时/日度/周度/月度数据
- 过期数据需重新采集

**异常值检测**：
- 与历史数据对比
- 识别离群值
- 确认或排除异常

### 5. Context 管理策略

**长任务隔离**：
```
主会话: 协调和汇总
├── Subagent 1: 数据采集A
├── Subagent 2: 数据采集B
├── Subagent 3: 数据验证
└── Subagent 4: 报告生成
```

**上下文清理**：
- 审核后清理中间过程
- 保留关键决策和结论
- 避免token浪费

## 📋 标准执行流程

### 数据采集任务模板

```bash
# 1. 创建多个Subagent并行采集
for source in source1 source2 source3; do
  sessions_spawn \
    --task "从$source采集[具体数据]" \
    --runtime "subagent" \
    --mode "run" \
    --timeoutSeconds 180
done

# 2. 等待所有Subagent完成
# 3. 中孚审核（验证数据质量）
# 4. 生成最终报告
```

### 代码生成任务模板

```bash
# Phase 1: Explore
# - 读取现有代码结构
# - 理解依赖关系
# - 识别复用模块

# Phase 2: Plan
# 输出执行计划：
# 1. 创建文件A - 负责X功能
# 2. 创建文件B - 负责Y功能
# 3. 修改文件C - 添加Z接口

# Phase 3: Code
# - 按规划逐步执行
# - 每步验证编译/运行
# - 小步提交
```

## 🛠️ 工具使用策略

### 搜索工具优先级

```
web_search   → 关键词搜索，无确切URL时（最轻）
web_fetch    → 已知URL，获取静态内容（中等）
browser      → JS渲染/登录状态/交互（最重）
```

### 文件操作最佳实践

**读取策略**：
- 先搜索定位，再精确读取
- 大文件使用offset/limit分页
- 避免一次性读取超大文件

**编辑策略**：
- 使用edit工具精确替换
- 避免大范围重写
- 保持代码风格一致

### 执行命令安全原则

**必须询问**：
- 删除文件（使用trash替代rm）
- 修改系统配置
- 发送网络请求到外部
- 涉及敏感数据的操作

**安全替代**：
- `trash` > `rm`
- `cp`备份 > 直接覆盖
- 测试环境 > 生产环境

## ⚠️ 常见陷阱

1. **上下文污染**：长任务不隔离，导致后续任务受干扰
2. **过早优化**：未充分探索就开始编码
3. **单点故障**：依赖单一数据源，无交叉验证
4. **token浪费**：不清理中间过程，导致上下文膨胀
5. **验证缺失**：执行后不验证结果，错误未被发现
