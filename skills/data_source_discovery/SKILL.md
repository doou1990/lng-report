## 发现流程

当某数据项连续 3 天为 D 级时，自动触发数据源发现流程：

### Step 1: 问题识别
- 识别连续 3 天 D 级的数据项
- 记录当前使用的数据源及失败原因
- 评估数据重要性（是否为核心 19 项）

### Step 2: 替代源搜索
使用以下策略搜索替代数据源：

**搜索关键词模板：**
```
{数据项名称} + "价格"/"数据" + "今日"/"实时" + 数据源类型
```

**数据源类型优先级：**
1. 官方 API（A 级潜力）
2. 交易所/交易中心（B 级潜力）
3. 行业权威网站（B/C 级潜力）
4. 行业公众号（C 级潜力）
5. 综合信息平台（C/D 级潜力）

### Step 3: 可用性验证
对新发现的数据源进行 3 天测试采集：

| 验证项 | 通过标准 | 权重 |
|--------|----------|------|
| 数据可获取性 | 3/3 天成功获取 | 30% |
| 数据时效性 | 当日或 T-1 数据 | 25% |
| 数据准确性 | 与现有源差异 < 10% | 25% |
| 数据稳定性 | 无频繁变动或异常 | 20% |

### Step 4: 质量评估
计算综合质量得分（0-100）：

```
质量得分 = 可获取性×30 + 时效性×25 + 准确性×25 + 稳定性×20
```

**等级判定：**
- 90-100: A 级（官方/API）
- 75-89: B 级（权威网站）
- 60-74: C 级（行业网站/公众号）
- <60: D 级（不可用）

### Step 5: 自动集成（C级以上）
如质量 ≥ C 级：
- 自动加入数据源矩阵
- 更新 SKILL.md 数据源配置
- 记录到本技能的历史记录
- 下次报告开始使用新源
