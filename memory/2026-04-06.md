### 1. Scrapling反爬绕过技术学习

**学习时间**: 09:38-10:00

**学习要点**:
- **Fetcher**: 基础获取，适合静态页面
- **StealthyFetcher**: 反爬虫模式，支持Cloudflare绕过
- **DynamicFetcher**: 完整浏览器自动化，支持JavaScript渲染
- **Adaptive Parsing**: 自适应解析，网站改版后仍可定位元素

**关键代码示例**:
```python
# 基础获取
from scrapling.fetchers import Fetcher
page = Fetcher.get('https://example.com')

# 反爬虫模式
from scrapling.fetchers import StealthyFetcher
StealthyFetcher.adaptive = True
page = StealthyFetcher.fetch('https://example.com', headless=True, solve_cloudflare=True)

# 浏览器自动化
from scrapling.fetchers import DynamicFetcher
page = DynamicFetcher.fetch('https://example.com', headless=True, network_idle=True)

# 自适应解析
items = page.css('.product', auto_save=True)  # 首次保存选择器
items = page.css('.product', adaptive=True)   # 改版后自动定位
```

**API逆向工程方法论**:
> "Web scraping is 80% reverse engineering"

关键步骤:
1. 打开DevTools Network tab
2. 过滤XHR/Fetch请求
3. 寻找JSON响应
4. 检查JS文件中的token生成逻辑
5. 使用cloudscraper绕过Cloudflare

### 3. Agent S3 - GUI Agent框架学习 (用户补充)

**学习时间**: 09:46-10:00

**仓库**: https://github.com/simular-ai/Agent-S

**核心定位**: Agent S是一个开源的计算机使用代理框架，使AI能够像人类一样操作计算机。

**版本演进**:
| 版本 | OSWorld得分 | 关键特性 | 时间 |
|------|-------------|----------|------|
| Agent S1 | 20.6% | 基础框架 | 2024-10 |
| Agent S2 | 48.8% | 分层Manager-Worker架构 | 2025-03 |
| **Agent S3** | **69.9%** (单run) / **72.6%** (bBoN) | **简化架构+原生Coding Agent+Behavior Best-of-N** | 2025-10 |

**人类水平**: 72% (Agent S3已超越)

**核心创新 - Behavior Best-of-N (bBoN)**:
```
传统方法: 单run执行，错误会累积，方差大
Agent S3: 多run并行 → 生成Facts → Behavior Narrative → Judge选择最优
```

**技术架构**:
1. **Facts生成**: 将原始agent run转换为简洁的facts陈述
2. **Behavior Narrative**: 串联facts形成行为叙事
3. **BJudge (Behavior Judge)**: 基于facts比较不同run，选择最优

**关键成果**:
- OSWorld: 69.9% (GPT-5 + bBoN) → 接近人类水平72%
- WindowsAgentArena: 50.2% → 56.6% (3 rollouts)
- AndroidWorld: 68.1% → 71.6% (3 rollouts)
- Judge与人类偏好对齐度: 92.8%

**架构简化**:
- Agent S2: 分层Manager-Worker架构（复杂）
- Agent S3: 移除层级，引入原生Coding Agent，性能提升13%

**推荐配置**:
```bash
# 主模型 + Grounding模型分离
--provider openai --model gpt-5-2025-08-07  # 主模型
--ground_provider huggingface --ground_model ui-tars-1.5-7b  # Grounding模型
```

### 5. XCrawl Skills - 网页采集技能框架 (用户补充)

**学习时间**: 09:56-10:00

**仓库**: https://github.com/xcrawl-api/xcrawl-skills

**核心定位**: XCrawl是一个网页数据基础设施产品，提供搜索、爬取、URL映射和站点爬取功能。该仓库提供生产导向的技能定义，帮助代理一致地调用XCrawl API。

**技能目录**:
| 技能 | 用途 |
|------|------|
| `xcrawl` | 默认入口，直接查找和单URL提取 |
| `xcrawl-scrape` | 单URL提取和结构化数据工作流 |
| `xcrawl-map` | 站点URL发现和范围规划 |
| `xcrawl-crawl` | 批量站点爬取和异步结果处理 |
| `xcrawl-search` | 基于查询的发现，支持位置/语言控制 |

**API端点**:
- Base URL: `https://run.xcrawl.com`
- `POST /v1/scrape` - 单页提取
- `POST /v1/map` - URL映射
- `POST /v1/crawl` - 批量爬取
- `POST /v1/search` - 搜索

**配置方式**:
```json
// ~/.xcrawl/config.json
{
  "XCRAWL_API_KEY": "<your_api_key>"
}
```

**使用示例**:
```bash
# 同步爬取
curl -sS -X POST "https://run.xcrawl.com/v1/scrape" \
  -H "Authorization: Bearer ${API_KEY}" \
  -d '{"url":"https://example.com","mode":"sync","output":{"formats":["markdown","links"]}}'
```

**可应用于LNG系统的启示**:

4. **多格式输出**: markdown、json、links等多种格式，满足不同场景
